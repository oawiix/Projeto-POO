> [!NOTE]
> **Objetivo:** O cliente que será atendido é uma gráfica onde são impressos banner, panfleto, cartão, lonas, entre outros produtos. O cliente nos procurou para resolver um problema muito
> individual dele. O cliente deseja realizar o Gerenciamento dos pedidos dele para facilitar a comunicação entre os colaboradores da empresa. Portanto, ele gostaria que você desenvolvesse uma solução elegante onde alguns requisitos fossem satisfeitos.
> 
> Lembrando que nesse projeto teremos vários requisitos levantados na entrevista, porém nem todos necessariamente são relevantes ao seu sistema. Tome as decisões de analista de sistemas e construa um diagrama de classes que atenda a necessidade de gerenciar os pedidos.

**Requisitos Funcionais do Projeto**

Requisitos Funcionais

• Pedido 

RF1 – O sistema deve buscar pedido

RF2 – O sistema deve adicionar pedido

RF3 – O sistema deve cancelar pedido

RF4 – O sistema deve permitir cadastrar pedido

RF5 - O sistema deve criar código do pedido

RF6 – O sistema deve permitir a alteração da situação do pedido

• Histórico do Pedido (empresa)

RR11 - O sistema deve permitir a visualização dos pedidos conforme o filtro

• Cadastro

RF7 – O sistema deve ter página de cadastrar cliente (nome, telefone, logradouro)

RF8 – O sistema deve criar código cliente

• Execução

RF9 – O sistema deve buscar cliente por atributos (administrador)

**BÔNUS CASO CONCLUIRMOS O PRIMORDIAL:**

RF10 - O sistema deve enviar mensagens pelo telefone do cliente sobre a situação do 
pedido (sistema)

RF11 - O sistema deve enviar uma mensagem de feedback após conclusão do pedido 
ao cliente

> [!WARNING]
> PÁGINAS ESSENCIAIS PARA A EXECUÇÃO DO PROJETO: PEDIDO E CADASTRO DO CLIENTE 

> [!IMPORTANT]
>  **Implementação:** A linguagem escolhida foi Java e o tomcat como servidor web java.  O colaborador poderá incluir um novo pedido ao sistema que será enviado a produção. Por fim, o sistema deve apresentar uma lista de pedidos com o status do pedido mostrando quais que estão na fila para serem confeccionados.
